5. ROUTING BUG FIX: "DISCOVERY" -> "DISCOVER"
Issue: The "Discover" link on the navbar and hero button were throwing a 404 Page Not Found error.
Cause: The links pointed to `/discover`, but the actual folder was named `/discovery`.
Fix: 
- Renamed the Next.js route folder: `src/app/discovery` -> `src/app/discover`
- Updated all internal links across the codebase (nav, hero, footer, etc.) to point to `/discover` instead of `/discovery` to accurately match the new route.

6. NAVBAR GLASSMORPHISM & SPACING
- Increased transparency: Changed `bg-background/80` to `bg-background/40` on the navbar background to allow the `backdrop-blur-xl` to create a much stronger iPhone-style frosted glass effect.
- Reduced bottom padding: Changed `py-4` to `pt-4 pb-2.5` to shave exactly 6px off the bottom of the navbar, tightening the header vertically.

7. MASTERPIECE TYPOGRAPHY
- Removed all gradients, translucent overlays, and background clip bugs that were causing clipping artifacts.
- Applied a strict, solid `text-zinc-900` (almost pure black) in Light Mode, and `text-zinc-300` in Dark Mode, so the text punches heavily off the background image using a strong `drop-shadow`.

8. HERO LAYOUT: SEAMLESS FULL-WIDTH
- Removed the top padding (`pt-2`, etc.) from the Hero container so it now sits perfectly flush under the glassy navbar.
- Removed the curved borders (`rounded-[3rem]`) and side margins/padding.
- Stretch to edge: The hero section now spans 100% of the screen width for a massive, immersive entryway instead of looking like a floating capsule.

9. SEAMLESS HERO TRANSITION
- Removed the hard bottom margin from the hero container.
- Added an absolute positioned gradient at the bottom of the hero image (`bg-gradient-to-t from-background to-transparent`) that perfectly blends the bottom edge of the photo into the background color of the next section, creating a seamless scroll transition instead of a hard cut.
